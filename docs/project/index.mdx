## 정리 기준

- **문제:** 어떤 반복 작업, 사용성 문제, 운영상의 비효율을 해결하려 했는지
- **역할:** 직접 설계하고 구현한 범위와 의사결정
- **접근:** 기술 선택, 데이터 흐름, 배포 방식에서의 판단
- **공개 범위:** 진행 중인 private 프로젝트는 코드 링크 없이 안전한 수준의 구조, 화면, 배운 점만 공개
- **다음 단계:** 제품 완성도를 높이기 위해 남겨둔 개선 항목

## 보류한 항목

`KeywordFinder`는 민감정보 정리 이력이 있어 공개 소개에서 제외했습니다. `allndo`, `firstlove_viewer`는 제품 방향과 README 정리가 더 필요해 이번 포트폴리오 반영에서는 보류했습니다.
